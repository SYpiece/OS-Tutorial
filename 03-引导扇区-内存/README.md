本课的唯一目标是学习引导扇区的存储位置

我可以直接告诉您BIOS将其放置在 `0x7C00` 地址，但通过错误示例能让原理更清晰。

我们要尝试在屏幕打印字母X。将使用4种不同策略，观察哪些有效及其原因：

**打开文件 `boot_sect_memory.asm`**

首先用标签定义X的数据：
```nasm
the_secret:
    db "X"
```

然后尝试多种方式访问 `the_secret`：

1. `mov al, the_secret`
2. `mov al, [the_secret]`
3. `mov al, the_secret + 0x7C00`
4. `mov al, 2d + 0x7C00`（其中 `2d` 是二进制中X字节的实际位置）

查看代码及注释说明。

编译运行后应显示类似 `1[2¢3X4X` 的字符串，其中1和2后的字节为随机乱码。

若增删指令，需重新计算X的偏移量（按字节计数），并替换 `0x2d` 为新值。

请务必100%理解引导扇区偏移和内存寻址后再继续。


全局偏移量
-----------------

由于处处加 `0x7c00` 极不方便，汇编器允许用 `org` 命令定义全局偏移：

```nasm
[org 0x7c00]
```

打开 `boot_sect_memory_org.asm` 文件，您将看到使用 `org` 指令的标准引导扇区数据打印方式（对应之前的尝试2）。编译运行后，观察 `org` 命令对各解决方案的影响。

阅读注释可完整理解有无 `org` 指令时的变化。
