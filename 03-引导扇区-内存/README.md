*在开始之前你可能需要 Google 了解的概念：内存偏移、指针*

**目标：理解计算机内存是如何组织的**

请先打开[本文档](附加文件/os-dev.pdf)<sup>1</sup>的第14页，查看内存布局示意图。

这个章节的主要目的是要明白，启动引导扇区的程序被储存在内存的哪个位置。

我可以直接告诉你，BIOS 会把引导程序放置在内存地址 `0x7C00` 处，然后把系统控制权交给这里的程序。要理解这一点，你最好是自己尝试一下别的位置。

我们要尝试在屏幕打印字母X。将使用4种不同策略，观察哪些有效及其原因。

**打开文件 `boot_sect_memory.asm`**

首先，我们定义了 X 是纯数据：
```nasm
the_secret:
    db "X"
```

然后尝试用多种方式访问 `the_secret`：

1. `mov al, the_secret`
2. `mov al, [the_secret]`
3. `mov al, the_secret + 0x7C00`
4. `mov al, 2d + 0x7C00`（其中 `2d` 是二进制中X字节的实际位置）

查看代码及注释说明。

编译并运行我们的代码。你可能会看到类似 `1[2¢3X4X` 的字符串，这里 1 和 2 后面的数据是随机的内存数据 。

如果你添加或者删除了一些指令，那么一定要记得重新计算 X 的偏移地址。然后用新的偏移地址代替代码中的 `0x2d` 。

请务必100%理解引导扇区偏移和内存寻址后再继续。


全局偏移量
-----------------

由于处处加 `0x7c00` 极不方便，汇编器允许你用 `org` 命令定义全局偏移：

```nasm
[org 0x7c00]
```

打开 `boot_sect_memory_org.asm` 文件，你会看到使用 `org` 指令的标准引导扇区数据打印方式，这个方式对应之前的尝试2。编译运行后，观察 `org` 命令对各解决方案的影响。

阅读注释可完整理解有无 `org` 指令时的变化。
