字符串
-------

像定义字节一样定义字符串，但要以空字节结尾（类似C语言）来确定字符串结束：

```nasm
mystring:
    db 'Hello, World', 0
```

注意：引号内的文本会被汇编器转换为ASCII码，末尾单独的0会被编译为 `0x00`（空字节）


控制结构
------------------

我们已用过一个：`jmp $`（无限循环）。

汇编跳转基于 *前一条* 指令的结果。例如：

```nasm
cmp ax, 4      ; 如果ax等于4
je ax_is_four  ; 执行特定操作（跳转到标签）
jmp else       ; 否则执行其他操作
jmp endif      ; 最后恢复主流程

ax_is_four:
    .....
    jmp endif

else:
    .....
    jmp endif  ; 非必须但为完整性保留

endif:
```

先用高级语言思维设计逻辑，再转换成这种汇编结构。

`jmp` 有多种条件变体：相等/小于等。这些指令很直观，可自行搜索具体用法。


函数调用
-----------------

函数调用本质上就是跳转到标签。

难点在于参数传递，需遵循两个步骤：

1. 程序员约定使用特定寄存器或内存地址传递参数
2. 编写更通用的代码，避免副作用

第一步很简单，我们约定使用 `al`（实际用 `ax`）传递参数：

```nasm
mov al, 'X'
jmp print
endprint:

...

print:
    mov ah, 0x0e  ; 终端模式
    int 0x10      ; 假设字符已在'al'中
    jmp endprint  ; 标签需预先约定
```

你会发现这种方法会导致代码耦合。当前print函数只能返回endprint，如果其他函数想要调用它怎么办？我们正在扼杀代码的重用性。

正确解决方案包含两个改进：
- 存储可变返回地址
- 保存寄存器状态以允许子函数修改寄存器

CPU通过 `call` 和 `ret` 指令自动存储返回地址。用 `pusha` 和 `popa` 指令可自动保存和恢复所有寄存器。
