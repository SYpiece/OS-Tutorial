*在开始之前你可能需要 Google 了解的概念：汇编器、BIOS*

**目标：创建一个BIOS能识别为可启动磁盘的文件**

这非常令人兴奋，我们要创建自己的引导扇区！

理论
----

当计算机启动时，因为BIOS不知道如何加载操作系统，所以它将这项任务交给引导扇区。因此，引导扇区必须位于已知的标准位置。这个位置就是磁盘的第一个扇区（柱面 0，磁头 0，扇区 0），占用512字节。

为了检查 "磁盘是可引导的"，BIOS会检查引导扇区的第511和512字节是不是为 `0xAA55`。

下面是有史以来最简单的引导扇区：

```
e9 fd ff 00 00 00 00 00 00 00 00 00 00 00 00 00
00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
[ 还有29行全零字节 ]
00 00 00 00 00 00 00 00 00 00 00 00 00 00 55 aa
```

它基本上全是零，以16位值 `0xAA55` 结尾（注意字节序，x86 是小端序）。开头的三个字节表示一个无限循环。

最简单的引导扇区
-------------------------

你可以用二进制编辑器编写上述512字节，或者编写一个非常简单的汇编代码：

```nasm
; 无限循环 (e9 fd ff)
loop:
    jmp loop 

; 用510个零填充，减去前面代码的大小
times 510-($-$$) db 0
; 魔术数字
dw 0xaa55 
```

编译：

`nasm -f bin boot_sect_simple.asm -o boot_sect_simple.bin`

> OSX提醒：如果出现错误，请重读第00章

我知道你迫不及待想试试（我也是！），所以我们开始吧：

`qemu boot_sect_simple.bin`

> 在某些系统上，你可能需要运行 `qemu-system-x86_64 boot_sect_simple.bin`。如果出现SDL错误，尝试传递 --nographic 或者 --curses 参数给 qemu 程序。

你会看到一个窗口弹出，显示 "Booting from Hard Disk..."，然后就没有其他反应了。你最后一次看到无限循环还非常的兴奋是什么时候？ ;-)
