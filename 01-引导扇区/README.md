**目标：创建一个BIOS能识别为可启动磁盘的文件**

这非常令人兴奋，我们要创建自己的引导扇区！

理论
----

当计算机启动时，BIOS不知道如何加载操作系统，因此它将这项任务委托给引导扇区。因此，引导扇区必须位于已知的标准位置。这个位置就是磁盘的第一个扇区（柱面0，磁头0，扇区0），占用512字节。

为了确保"磁盘可启动"，BIOS会检查被声明为引导扇区的第511和512字节是否为`0xAA55`。

这是有史以来最简单的引导扇区：

```
e9 fd ff 00 00 00 00 00 00 00 00 00 00 00 00 00
00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
[ 还有29行全零字节 ]
00 00 00 00 00 00 00 00 00 00 00 00 00 00 55 aa
```

它基本上全是零，以16位值`0xAA55`结尾（注意字节序，x86是小端序）。前三个字节执行一个无限跳转。

有史以来最简单的引导扇区
-------------------------

您可以用二进制编辑器编写上述512字节，或者编写一个非常简单的汇编代码：

```nasm
; 无限循环 (e9 fd ff)
loop:
    jmp loop 

; 用510个零填充，减去前面代码的大小
times 510-($-$$) db 0
; 魔术数字
dw 0xaa55 
```

编译命令：
`nasm -f bin boot_sect_simple.asm -o boot_sect_simple.bin`

> OSX警告：如果出现错误，请重读第00章

我知道你迫不及待想试试（我也是！），所以我们开始吧：

`qemu boot_sect_simple.bin`

> 在某些系统上，您可能需要运行`qemu-system-x86_64 boot_sect_simple.bin`。如果出现SDL错误，尝试添加--nographic和/或--curses标志。
